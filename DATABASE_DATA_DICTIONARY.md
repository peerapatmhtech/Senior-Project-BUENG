# พจนานุกรมข้อมูลฐานข้อมูล (Database Data Dictionary)

เอกสารฉบับนี้รวบรวมโครงสร้างข้อมูล (Data Dictionary) ของทุกคอลเลกชัน (Collections) ที่ใช้งานจริงภายในระบบ Backend (อ้างอิงจากโฟลเดอร์ `src/model/`) โดยจัดทำในรูปแบบตารางเพื่อให้นำไปใช้งานประกอบการทำรูปเล่มปริญญานิพนธ์ได้ทันที

---

### 1. หมวดหมู่ผู้ใช้งานและการยืนยันตัวตน (Users & Authentication)

**1.1 Collection: Gmail (ตารางเก็บข้อมูลบัญชีผู้ใช้งานพื้นฐาน)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อจัดเก็บข้อมูลพื้นฐานของผู้ใช้งานที่สมัครสมาชิกและเข้าสู่ระบบด้วย Google Account (Gmail) โดยจะเก็บข้อมูลที่จำเป็นต่อการยืนยันตัวตน เช่น อีเมล ชื่อแสดงผล รูปภาพโปรไฟล์เบื้องต้น และสถานะการยืนยันตัวตน เพื่อใช้เป็นรหัสอ้างอิงหลัก (Primary Key) ในการผูกข้อมูลกับระบบส่วนอื่นๆ_

| Collection | seq | Field Name               |  Format  | Description (คำอธิบาย)                   |
| :--------- | :-: | :----------------------- | :------: | :--------------------------------------- |
| **Gmail**  |  1  | `_id`                    | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|            |  2  | `email`                  |  String  | อีเมลของผู้ใช้งาน (Unique)               |
|            |  3  | `displayName`            |  String  | ชื่อแสดงผลของผู้ใช้งาน                   |
|            |  4  | `photoURL`               |  String  | URL รูปภาพโปรไฟล์                        |
|            |  5  | `photosOrder`            |  Array   | ลำดับการจัดเรียงรูปภาพโปรไฟล์            |
|            |  6  | `isVerified`             | Boolean  | สถานะการยืนยันตัวตนผ่านอีเมล             |
|            |  7  | `emailVerificationToken` |  String  | รหัส Token สำหรับยืนยันอีเมล             |
|            |  8  | `createdAt`              |   Date   | เวลาที่สร้างข้อมูล                       |
|            |  9  | `updatedAt`              |   Date   | เวลาที่แก้ไขข้อมูลล่าสุด                 |

**1.2 Collection: Info (ตารางเก็บข้อมูลโปรไฟล์และรายละเอียดเชิงลึกของผู้ใช้งาน)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อจัดเก็บข้อมูลส่วนตัวที่ผู้ใช้งานกรอกเพิ่มเติม (Profile) เช่น ชื่อเล่น ข้อความแนะนำตัว และรายละเอียดอื่นๆ เพื่อใช้แสดงผลบนหน้าโปรไฟล์และเป็นข้อมูลประกอบในการจับคู่หากิจกรรมและหาเพื่อนที่สนใจในสิ่งเดียวกันได้อย่างแม่นยำยิ่งขึ้น_

| Collection | seq | Field Name             |  Format  | Description (คำอธิบาย)                   |
| :--------- | :-: | :--------------------- | :------: | :--------------------------------------- |
| **Info**   |  1  | `_id`                  | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|            |  2  | `email`                |  String  | อีเมลที่ใช้อ้างอิงผู้ใช้งาน (Unique)     |
|            |  3  | `nickname`             |  String  | ชื่อเล่นของผู้ใช้งาน                     |
|            |  4  | `userInfo.detail`      |  String  | ข้อความแนะนำตัวหรือรายละเอียด About Me   |
|            |  5  | `userInfo.description` |  String  | คำอธิบายเพิ่มเติมเกี่ยวกับผู้ใช้งาน      |
|            |  6  | `userInfo.extra`       |  String  | ข้อมูลเสริมอื่นๆ                         |
|            |  7  | `joinedRooms`          |  Array   | รายการห้องแชท/ชุมชนที่เข้าร่วมแล้ว       |
|            |  8  | `joinedRooms.roomId`   |  String  | รหัสห้องแชท                              |
|            |  9  | `joinedRooms.roomName` |  String  | ชื่อห้องแชท                              |
|            | 10  | `createdAt`            |   Date   | เวลาที่สร้างข้อมูล                       |
|            | 11  | `updatedAt`            |   Date   | เวลาที่แก้ไขข้อมูลล่าสุด                 |

**1.3 Collection: Filter (ตารางเก็บข้อมูลตัวกรองความสนใจของผู้ใช้งาน)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อจัดเก็บหมวดหมู่ความสนใจหลัก (Genres) และหมวดหมู่ย่อย (SubGenres) ที่ผู้ใช้งานได้เลือกไว้ โดยระบบจะนำข้อมูลในตารางนี้ไปใช้เป็นคีย์เวิร์ด (Keyword) ตั้งต้นในการค้นหากิจกรรมที่เหมาะสม ทั้งการค้นหาจากฐานข้อมูลภายใน (Local DB) และการส่งคำสั่งค้นหาไปยังบริการภายนอก (SerpApi) เพื่อให้ได้ผลลัพธ์ที่ตอบโจทย์ความสนใจของผู้ใช้แต่ละคนมากที่สุด_

| Collection | seq | Field Name  |  Format  | Description (คำอธิบาย)                       |
| :--------- | :-: | :---------- | :------: | :------------------------------------------- |
| **Filter** |  1  | `_id`       | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key)     |
|            |  2  | `email`     |  String  | อีเมลที่ใช้อ้างอิงผู้ใช้งาน                  |
|            |  3  | `genres`    |  Array   | รายการหมวดหมู่หลักที่สนใจ (Array of Strings) |
|            |  4  | `subGenres` |   Map    | เก็บรายการหมวดหมู่ย่อยแบบ Map Object         |
|            |  5  | `createdAt` |   Date   | เวลาที่สร้างข้อมูล                           |
|            |  6  | `updatedAt` |   Date   | เวลาที่แก้ไขข้อมูลล่าสุด                     |

**1.4 Collection: UserPhoto (ตารางเก็บรูปภาพเพิ่มเติมในแกลเลอรีโปรไฟล์ของผู้ใช้)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อจัดเก็บรูปภาพเพิ่มเติมที่ผู้ใช้อัปโหลดเข้ามาในแกลเลอรีส่วนตัว โดยแยกรักษาข้อมูล URL และไฟล์รูปภาพออกจากข้อมูลบัญชีหลัก เพื่อให้ง่ายต่อการเรียกดู จัดการ และประหยัดพื้นที่โหลดข้อมูล (Bandwidth) เมื่อมีการเรียกดูข้อมูลโปรไฟล์ทั่วไป_

| Collection    | seq | Field Name  |  Format  | Description (คำอธิบาย)                   |
| :------------ | :-: | :---------- | :------: | :--------------------------------------- |
| **UserPhoto** |  1  | `_id`       | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|               |  2  | `email`     |  String  | อีเมลเจ้าของรูปภาพ                       |
|               |  3  | `url`       |  String  | URL สำหรับเข้าถึงรูปภาพ                  |
|               |  4  | `filename`  |  String  | ชื่อไฟล์ของรูปภาพ                        |
|               |  5  | `title`     |  String  | คำอธิบายรูปภาพ (ถ้ามี)                   |
|               |  6  | `createdAt` |   Date   | เวลาที่อัปโหลดรูปภาพ                     |

### 2. หมวดหมู่กิจกรรม (Events)

**2.1 Collection: Event (ตารางเก็บข้อมูลรายละเอียดกิจกรรม)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อจัดเก็บข้อมูลกิจกรรมทั้งหมดที่แสดงผลในระบบ ทั้งข้อมูลกิจกรรมที่มีอยู่เดิมในฐานข้อมูลและกิจกรรมใหม่ที่ดึงมาจาก SerpApi ข้อมูลประกอบด้วยชื่อกิจกรรม วันที่ สถานที่จัดงาน รูปภาพ และลิงก์ไปยังต้นฉบับ เพื่อใช้เป็นข้อมูลหลักสำหรับแสดงผลและให้ผู้ใช้งานกดไลก์หรือจับคู่_

| Collection | seq | Field Name           |  Format  | Description (คำอธิบาย)                          |
| :--------- | :-: | :------------------- | :------: | :---------------------------------------------- |
| **Event**  |  1  | `_id`                | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key)        |
|            |  2  | `title`              |  String  | ชื่อกิจกรรม                                     |
|            |  3  | `date`               |  Mixed   | วันที่จัดกิจกรรม (รองรับหลายรูปแบบ)             |
|            |  4  | `email`              |  String  | อีเมลผู้ดึงข้อมูลเข้า (อ้างอิง)                 |
|            |  5  | `address`            |  Mixed   | ที่อยู่สถานที่จัดงาน                            |
|            |  6  | `genre`              |  Mixed   | หมวดหมู่กิจกรรม                                 |
|            |  7  | `description`        |  String  | รายละเอียดกิจกรรม                               |
|            |  8  | `link`               |  String  | URL ลิงก์ไปยังหน้าเว็บกิจกรรม                   |
|            |  9  | `image`              |  String  | URL รูปภาพประกอบหลัก                            |
|            | 10  | `thumbnail`          |  String  | URL รูปภาพขนาดเล็ก                              |
|            | 11  | `venue`              |  Mixed   | ข้อมูลสถานที่จัดงาน                             |
|            | 12  | `ticket_info`        |  Mixed   | ข้อมูลการขายบัตร/เข้าร่วม                       |
|            | 13  | `event_location_map` |  Mixed   | ข้อมูลแผนที่ Google Maps                        |
|            | 14  | `createdByAI`        | Boolean  | แฟล็กแสดงว่าดึงข้อมูลอัตโนมัติ (Defalut: false) |
|            | 15  | `createdAt`          |   Date   | เวลาที่บันทึกข้อมูล                             |
|            | 16  | `updatedAt`          |   Date   | เวลาที่อัปเดตข้อมูล                             |

**2.2 Collection: UserEvent (ตารางเก็บความสัมพันธ์ระหว่างผู้ใช้กับกิจกรรม)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อจัดเก็บสถานะความสัมพันธ์ว่าผู้ใช้งานแต่ละคนมีการโต้ตอบกับกิจกรรมใดบ้าง (เช่น การกดซ่อนกิจกรรม หรือเข้าร่วมกิจกรรม) เพื่อเป็นข้อมูลให้ระบบใช้ในการคัดกรองหรือเลือกแสดงผลกิจกรรมที่เหมาะสมบนหน้าฟีดของผู้ใช้_

| Collection    | seq | Field Name  |  Format  | Description (คำอธิบาย)                   |
| :------------ | :-: | :---------- | :------: | :--------------------------------------- |
| **UserEvent** |  1  | `_id`       | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|               |  2  | `email`     |  String  | อีเมลของผู้ใช้งาน                        |
|               |  3  | `eventId`   | ObjectId | รหัสกิจกรรม (อ้างอิงไปยังตาราง Event)    |
|               |  4  | `status`    |  String  | สถานะ (active, deleted, archived)        |
|               |  5  | `createdAt` |   Date   | เวลาที่สร้างข้อมูล                       |
|               |  6  | `updatedAt` |   Date   | เวลาที่อัปเดตข้อมูล                      |

**2.3 Collection: Like (ตารางเก็บข้อมูลการกดถูกใจกิจกรรม)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อบันทึกข้อมูลว่าผู้ใช้งานกรุณากดถูกใจ (Like) กิจกรรมใดไปบ้าง โดยข้อมูลนี้มีความสำคัญมากในการนำไปใช้วิเคราะห์และจับคู่ระหว่างผู้ใช้งานที่สนใจเข้าร่วมกิจกรรมเดียวกัน (Event-based Matching)_

| Collection | seq | Field Name   |  Format  | Description (คำอธิบาย)                   |
| :--------- | :-: | :----------- | :------: | :--------------------------------------- |
| **Like**   |  1  | `_id`        | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|            |  2  | `userEmail`  |  String  | อีเมลของผู้กดถูกใจ                       |
|            |  3  | `eventId`    | ObjectId | รหัสกิจกรรมที่ถูกใจ (อ้างอิง Event)      |
|            |  4  | `eventTitle` |  String  | ชื่อกิจกรรมที่ถูกใจ                      |
|            |  5  | `likedAt`    |   Date   | เวลาที่กดถูกใจ                           |

### 3. หมวดหมู่การจับคู่และหาเพื่อน (Matching & Friends)

**3.1 Collection: InfoMatch (ตารางคำนวณและเก็บข้อมูลการจับคู่หาเพื่อน)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อประมวลผลและจัดเก็บข้อมูลเปอร์เซ็นต์ความเข้ากันได้ (Matching Score) ระหว่างผู้ใช้งานสองคน โดยพิจารณาจากกิจกรรมที่สนใจตรงกัน โดยจะใช้เป็นหลักฐานในการตัดสินใจว่าหากทั้งสองฝ่ายกดไลก์ซึ่งกันและกัน จะทำการแมทช์เป็นเพื่อนกันอย่างสมบูรณ์ต่อไป_

| Collection    | seq | Field Name       |  Format  | Description (คำอธิบาย)                      |
| :------------ | :-: | :--------------- | :------: | :------------------------------------------ |
| **InfoMatch** |  1  | `_id`            | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key)    |
|               |  2  | `eventId`        | ObjectId | รหัสกิจกรรมที่จับคู่ (ถ้ามี)                |
|               |  3  | `detail`         |  String  | รายละเอียด/ชื่อกิจกรรมที่ตรงกัน             |
|               |  4  | `email`          |  String  | อีเมลผู้ใช้ A                               |
|               |  5  | `usermatch`      |  String  | อีเมลผู้ใช้ B ที่ถูกนำมาจับคู่              |
|               |  6  | `chance`         |  Number  | เปอร์เซ็นต์ความเข้ากันได้ (Matching Score)  |
|               |  7  | `status`         |  String  | สถานะการแมทช์ (pending, matched, unmatched) |
|               |  8  | `initiatorEmail` |  String  | ผู้ที่เริ่มกดไลก์หรือจับคู่ก่อน             |
|               |  9  | `skipCount`      |  Number  | จำนวนครั้งที่ถูกปัดข้าม                     |
|               | 10  | `createdAt`      |   Date   | เวลาที่คำนวณการจับคู่                       |
|               | 11  | `updatedAt`      |   Date   | เวลาที่อัปเดตข้อมูลล่าสุด                   |

**3.2 Collection: FriendRequest (ตารางเก็บประวัติการส่งคำขอเป็นเพื่อน)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อบันทึกประวัติการส่งคำขอเป็นเพื่อน (Friend Request) ระหว่างผู้ใช้ โดยเก็บสถานะว่าคำขอนั้นกำลังรอดำเนินการ, ถูกตอบรับ, หรือถูกปฏิเสธ เพื่อใช้ในการจัดการแจ้งเตือน (Notifications) และควบคุมสิทธิ์การเกิดความสัมพันธ์ภายในระบบ_

| Collection        | seq | Field Name         |  Format  | Description (คำอธิบาย)                   |
| :---------------- | :-: | :----------------- | :------: | :--------------------------------------- |
| **FriendRequest** |  1  | `_id`              | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|                   |  2  | `requestId`        |  String  | รหัสอ้างอิงคำขอ (Unique)                 |
|                   |  3  | `from.email`       |  String  | อีเมลผู้ส่งคำขอ                          |
|                   |  4  | `from.displayName` |  String  | ชื่อผู้ส่งคำขอ                           |
|                   |  5  | `from.photoURL`    |  String  | รูปผู้ส่งคำขอ                            |
|                   |  6  | `to`               |  String  | อีเมลผู้รับคำขอ                          |
|                   |  7  | `status`           |  String  | สถานะ (pending, accepted, declined)      |
|                   |  8  | `roomId`           |  String  | รหัสห้องแชทที่สร้างหลังเป็นเพื่อนกันแล้ว |
|                   |  9  | `read`             | Boolean  | สถานะการอ่านแจ้งเตือน                    |
|                   | 10  | `timestamp`        |   Date   | เวลาที่กระทำรายการ (Default: Date.now)   |
|                   | 11  | `createdAt`        |   Date   | เวลาที่สร้างข้อมูลในระบบ                 |
|                   | 12  | `updatedAt`        |   Date   | เวลาที่แก้ไขข้อมูลล่าสุด                 |

**3.3 Collection: Friend (ตารางเก็บรายชื่อเพื่อนและห้องแชตคู่)**
_   _

| Collection | seq | Field Name       |  Format  | Description (คำอธิบาย)                     |
| :--------- | :-: | :--------------- | :------: | :----------------------------------------- |
| **Friend** |  1  | `_id`            | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key)   |
|            |  2  | `email`          |  String  | อีเมลเจ้าของรายการเพื่อน                   |
|            |  3  | `friends`        |  Array   | รายชื่อเพื่อนที่เพิ่มแล้ว                  |

|            |  5  | `friends.roomId` |  String  | รหัสห้องสนทนาส่วนตัว 1-1                   |
|            |  6  | `following`      |  Array   | รายชื่อคนที่กำลังติดตาม (Array of Strings) |
|            |  7  | `followers`      |  Array   | รายชื่อคนมาติดตาม (Array of Strings)       |
|            |  8  | `createdAt`      |   Date   | เวลาที่สร้างข้อมูล                         |
|            |  9  | `updatedAt`      |   Date   | เวลาที่แก้ไขข้อมูลล่าสุด                   |

**3.4 Collection: EventMatch (ตารางข้อมูลการจับคู่แบบเจาะจงกิจกรรม)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อรองรับข้อมูลทางลอจิกของการจับคู่ (Matching Logic) ในรูปแบบเก่าที่อิงตัวกิจกรรมเป็นหลัก โดยจัดเก็บความน่าจะเป็นในการจับคู่จากการตรวจสอบว่าอีกฝ่ายได้เข้าร่วมกิจกรรมที่สนใจตรงกันหรือไม่ (ปัจจุบันใช้เป็นข้อมูลอ้างอิงและประวัติ)_

| Collection     | seq | Field Name        |  Format  | Description (คำอธิบาย)                   |
| :------------- | :-: | :---------------- | :------: | :--------------------------------------- |
| **EventMatch** |  1  | `_id`             | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|                |  2  | `title`           |  String  | ชื่อเรื่อง/ชื่อกิจกรรม                   |
|                |  3  | `email`           |  String  | อีเมลผู้ใช้งานหลัก                       |
|                |  4  | `usermatchjoined` | Boolean  | ตรวจสอบว่าอีกฝ่ายเข้าร่วมหรือไม่         |
|                |  5  | `emailjoined`     | Boolean  | ตรวจสอบว่าตนเองเข้าร่วมหรือไม่           |
|                |  6  | `chance`          |  Number  | เปอร์เซ็นต์ความน่าจะเป็น                 |
|                |  7  | `usermatch`       |  String  | อีเมลผู้ใช้เป้าหมาย                      |
|                |  8  | `createdAt`       |   Date   | เวลาที่สร้างข้อมูล                       |
|                |  9  | `updatedAt`       |   Date   | เวลาที่แก้ไขข้อมูลล่าสุด                 |

### 4. หมวดหมู่ห้องสนทนาชุมชน (Community & Chat Rooms)

**4.1 Collection: Room (ตารางข้อมูลรายละเอียดห้องแชตชุมชน)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อบันทึกข้อมูลตั้งต้นของห้องชุมชน (Community/Club) เช่น ชื่อกลุ่ม รายละเอียด แท็กที่เกี่ยวข้อง และจำนวนสมาชิกรองรับสูงสุด เพื่อให้ผู้ใช้งานคนอื่นๆ สามารถค้นหาและเข้าร่วมพูดคุยแลกเปลี่ยนประสบการณ์ตามหมวดหมู่กิจกรรมที่สนใจร่วมกันได้_

| Collection | seq | Field Name    |  Format  | Description (คำอธิบาย)                   |
| :--------- | :-: | :------------ | :------: | :--------------------------------------- |
| **Room**   |  1  | `_id`         | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key) |
|            |  2  | `roomId`      |  String  | รหัสอ้างอิงห้องแบบข้อความ                |

|            |  5  | `description` |  String  | รายละเอียดจุดประสงค์ของห้องชุมชน         |
|            |  6  | `memberLimit` |  Number  | จำนวนสมาชิกรองรับสูงสุด (Min: 1)         |
|            |  7  | `tags`        |  Array   | แท็กหมวดหมู่ความสนใจ (Array of Strings)  |
|            |  8  | `createdBy`   |  String  | อีเมลผู้สร้างห้องชุมชน                   |
|            |  9  | `createdAt`   |   Date   | เวลาที่สร้างห้องชุมชน                    |

**4.2 Collection: UserRoom (ตารางสถานะผู้ใช้งานในระบบแชตและชุมชน)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อคอยติดตามความเคลื่อนไหว (Presence) ของผู้ใช้งานในระบบแชต ทั้งข้อมูลการออนไลน์แบบเรียลไทม์ (IsOnline), รอบการใช้งานครั้งล่าสุด (Last Seen), และรายชื่อเพื่อนที่ผูกกับห้องสนทนา เพื่อแสดงสัญลักษณ์วงกลมสีเขียวแจ้งสถานะให้คนอื่นๆ ทราบ_

| Collection   | seq | Field Name        |  Format  | Description (คำอธิบาย)                          |
| :----------- | :-: | :---------------- | :------: | :---------------------------------------------- |

|              |  2  | `email`           |  String  | อีเมลผู้ใช้งาน (Unique)                         |
|              |  3  | `displayName`     |  String  | ชื่อแสดงผล                                      |

|              |  5  | `friends`         |  Array   | รายการเพื่อนและห้องแชท (เพื่อนแบบ Array Object) |

**4.3 Collection: AiChatMessage (ตารางเก็บประวัติการสนทนากับระบบปัญญาประดิษฐ์)**
_ตารางนี้ถูกสร้างขึ้นมาเพื่อเก็บบันทึกซ้ำการโต้ตอบพูดคุยระหว่างผู้ใช้งานกับระบบผู้ช่วยอัจฉริยะ (AI Bot) ไว้ตรวจสอบย้อนหลัง แยกแยะผู้ส่งให้เห็นเป็นลักษณะแชตส่วนตัวกับ AI เพื่อพัฒนาบริบทของ AI ให้โต้ตอบกับคนๆ นั้นได้อย่างมีประสิทธิภาพมากขึ้นในอนาคต_
