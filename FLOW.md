InfoMatch Model นี้เป็นหัวใจสำคัญของระบบจับคู่ (Matching) ทำหน้าที่เหมือน "การ์ด" ที่บันทึกความสัมพันธ์ที่เป็นไปได้ระหว่างผู้ใช้ 2 คนที่สนใจในกิจกรรม (Event) เดียวกัน

Use Case หลักของ InfoMatch Model
1. การสร้าง "โอกาสในการจับคู่" (Creating a Potential Match)
สถานการณ์:
ผู้ใช้ A กด "Like" (หรือกด Favorite) ที่กิจกรรม "Concert A"
ระบบจะตรวจสอบว่ามีผู้ใช้อื่น (เช่น ผู้ใช้ B) ที่เคย "Like" กิจกรรม "Concert A" นี้ไว้ก่อนแล้วหรือไม่
การทำงานของ Model:
ถ้ามี, ระบบจะสร้างเอกสาร InfoMatch ใหม่ขึ้นมา 1 ชุด โดยมีข้อมูลดังนี้:
eventId: ID ของ "Concert A"
email: "userA@email.com"
usermatch: "userB@email.com"
status: pending (สถานะเริ่มต้น คือ "รอการตอบรับ")
initiatorEmail: "userA@email.com" (คนที่กดไลค์ทีหลังและทำให้เกิดการตรวจสอบนี้)
skipCount: 0
เอกสารนี้เปรียบเสมือนการ์ดที่รอให้ผู้ใช้ B เห็นและตัดสินใจในหน้า Tinder-like UI
2. การ "จับคู่สำเร็จ" (Completing a Match)
สถานการณ์:
ผู้ใช้ B เข้ามาที่หน้าจับคู่ และเห็นการ์ดของผู้ใช้ A (ที่สร้างจาก Use Case ข้อ 1)
ผู้ใช้ B สนใจและกด "Like" (หรือปัดขวา) ที่การ์ดของผู้ใช้ A
การทำงานของ Model:
ระบบจะค้นหาเอกสาร InfoMatch ที่มีอยู่ (ที่มี status เป็น pending)
จากนั้นจะอัปเดต status ของเอกสารนั้นจาก pending เป็น matched
เมื่อสถานะเป็น matched แล้ว:
Frontend จะแสดง Modal "It's a Match!"
ผู้ใช้ทั้งสองคนสามารถเริ่มแชทกันได้
การ์ดนี้จะไม่ปรากฏในหน้าจับคู่อีกต่อไป
3. การ "ปฏิเสธการจับคู่" (Skipping or Unmatching)
สถานการณ์:
ผู้ใช้ B เห็นการ์ดของผู้ใช้ A แต่ไม่สนใจ และกด "Skip" (หรือปัดซ้าย)
การทำงานของ Model:
ระบบจะค้นหาเอกสาร InfoMatch ที่เกี่ยวข้อง
จากนั้นจะเพิ่มค่า skipCount ขึ้น 1 (skipCount: 1)
Logic เพิ่มเติม: ถ้า skipCount ถึงค่าที่กำหนด (เช่น 3 ครั้ง) ระบบจะอัปเดต status เป็น unmatched
เมื่อสถานะเป็น unmatched การ์ดใบนี้จะถูกซ่อนและไม่แสดงให้ผู้ใช้ทั้งสองเห็นอีก เพื่อป้องกันการเจอคนเดิมซ้ำๆ ที่เราไม่สนใจ
สรุป Flow การเปลี่ยนแปลงของ status
สถานะของ InfoMatch จะเปลี่ยนไปตามการกระทำของผู้ใช้ ดังนี้:

pending (เริ่มต้น)
ถ้าผู้ใช้อีกฝ่ายกด Like: status -> matched
ถ้าผู้ใช้คนใดคนหนึ่งกด Skip: skipCount เพิ่มขึ้น และถ้าถึง 3 status -> unmatched
Model นี้ช่วยให้คุณสามารถจัดการสถานะความสัมพันธ์ระหว่างผู้ใช้ได้อย่างเป็นระบบและชัดเจน ตั้งแต่การพบกันครั้งแรกไปจนถึงการจับคู่สำเร็จหรือการปฏิเสธครับ
