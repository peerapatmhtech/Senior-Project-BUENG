**ตารางแสดงรายละเอียดพจนานุกรมข้อมูล (Data Dictionary) สำหรับ Profile (อ้างอิงจากคอลเลกชัน Info)**

ตารางด้านล่างนี้แสดงโครงสร้างการจัดเก็บข้อมูลส่วนตัวของผู้ใช้งาน (Profile) ในฐานข้อมูล ซึ่งอ้างอิงจาก Schema หลักที่ใช้ในการทำงานของระบบ (อิงจาก `infoSchema`) โดยมีการเพิ่มคอลัมน์และคำอธิบายให้ครบถ้วนเหมาะสมกับการนำไปใส่ในรูปเล่มโครงงานครับ:

| Collection         | seq | Field Name             |  Format  | Description (คำอธิบาย)                                          |
| :----------------- | :-: | :--------------------- | :------: | :-------------------------------------------------------------- |
| **Profile (Info)** |  1  | `_id`                  | ObjectId | รหัสอ้างอิงเอกสารอัตโนมัติ (Primary Key)                        |
|                    |  2  | `email`                |  String  | อีเมลที่ใช้ในการเข้าสู่ระบบและอ้างอิงตัวตนผู้ใช้งาน (Unique)    |
|                    |  3  | `nickname`             |  String  | ชื่อเล่นหรือนามแฝงที่ผู้ใช้งานตั้งค่าไว้สำหรับแสดงผล            |
|                    |  4  | `userInfo.detail`      |  String  | ข้อความแนะนำตัวหรือรายละเอียดเกี่ยวกับผู้ใช้งาน (About Me)      |
|                    |  5  | `userInfo.description` |  String  | คำอธิบายเพิ่มเติมเกี่ยวกับผู้ใช้งาน                             |
|                    |  6  | `userInfo.extra`       |  String  | ข้อมูลเสริมอื่นๆ ของผู้ใช้งาน                                   |
|                    |  7  | `joinedRooms`          |  Array   | รายการห้องสนทนาหรือกลุ่มชุมชนที่ผู้ใช้งานเข้าร่วมแล้ว           |
|                    |  8  | `joinedRooms.roomId`   |  String  | รหัสอ้างอิงของห้องสนทนาที่ผู้ใช้เข้าร่วม                        |
|                    |  9  | `joinedRooms.roomName` |  String  | ชื่อของห้องสนทนาที่ผู้ใช้เข้าร่วม                               |
|                    | 10  | `createdAt`            |   Date   | วันที่และเวลาที่สร้างประวัติข้อมูลผู้ใช้นี้ครั้งแรก (Timestamp) |
|                    | 11  | `updatedAt`            |   Date   | วันที่และเวลาที่มีการแก้ไขประวัติมูลล่าสุด (Timestamp)          |

---

**หมายเหตุสำหรับการนำไปใช้ในเล่ม:**

- คุณสามารถนำตารางนี้ไปสร้างใหม่ใน Microsoft Word เพื่อปรับขนาดคอลัมน์และจัดฟอนต์ (เช่น TH Sarabun PSK) ให้สวยงามตามมาตรฐานการทำรูปเล่มของมหาวิทยาลัยได้เลยครับ
- ฟิลด์ `createdAt` และ `updatedAt` ถูกสร้างอัตโนมัติจากฐานข้อมูล MongoDB (Mongoose `timestamps: true`) จึงควรใส่ไว้ในเล่มให้ครบถ้วนเพื่อความถูกต้องทางเทคนิคครับ
